# Configuration System

This directory contains the YAML-based configuration system for StickyNotes.

## Files

### `schema.js`

JSON Schema definition for config.yaml validation using ajv.

**Exports:**

- `configSchema` - JSON Schema object
- `validateConfig(config)` - Validate config object against schema
- `getDefaultValue(key)` - Get default value for a setting
- `getAllDefaults()` - Get all defaults as flat object
- `getAllDefaultsNested()` - Get all defaults as nested object
- `requiresRestart(key)` - Check if setting requires restart
- `getConstraints(key)` - Get validation constraints for a setting

**Example:**

```javascript
const { validateConfig } = require('./schema.js');

const result = validateConfig({
  schemaVersion: 2,
  general: { autoSaveDelay: 2000 },
});

if (!result.valid) {
  console.error('Validation errors:', result.errors);
}
```

### `defaults.js`

Default configuration generator with inline comments.

**Exports:**

- `generateDefaultConfig()` - Generate config.yaml with full documentation
- `generateMinimalConfig()` - Generate minimal config.yaml (no comments)
- `getSQLiteOnlySettings()` - List of settings that stay in SQLite
- `getYAMLSettings()` - List of settings that go to YAML
- `shouldMigrateToYAML(key)` - Check if a setting should migrate to YAML

**Example:**

```javascript
const { generateDefaultConfig, shouldMigrateToYAML } = require('./defaults.js');
const fs = require('fs');

// Generate config file on first run
const content = generateDefaultConfig();
fs.writeFileSync('config.yaml', content);

// Check if setting migrates to YAML
console.log(shouldMigrateToYAML('general.startMinimized')); // true
console.log(shouldMigrateToYAML('whisper.enabled')); // false
```

### `template.yaml`

Static reference template showing all available settings with documentation.
This is the same content as generated by `generateDefaultConfig()` but as a static file.

Use this for:

- Documentation reference
- IDE autocomplete in user's config.yaml
- Copy-paste when manually creating config files

### `manager.js`

ConfigManager class - Production-ready YAML configuration manager with hot reload, file locking, and event notifications.

**Features:**

- In-memory cache for synchronous reads (no I/O overhead)
- Hot reload with file watching (500ms debounce)
- Atomic writes using write-file-atomic
- File locking to prevent concurrent write conflicts
- Comment preservation when editing programmatically
- EventEmitter for change notifications
- Graceful error handling with corrupt file recovery
- Singleton pattern ensures one instance per config file

**Exports:**

- `ConfigManager` - Main class
- `getConfigManager(configPath)` - Singleton factory function
- `clearInstances()` - Clear singleton cache (for testing)

**Basic Usage:**

```javascript
const { getConfigManager } = require('./manager');
const path = require('path');

// Get singleton instance
const configPath = path.join(__dirname, 'config.yaml');
const config = getConfigManager(configPath);

// Get value (synchronous, uses cache)
const theme = config.get('appearance.theme');

// Set value (updates cache, schedules save)
config.set('appearance.theme', 'dark');

// Get all values
const all = config.getAll(); // Returns deep copy

// Save to disk (async)
await config.save();

// Start watching for external changes
config.watch();
config.on('configChanged', (changedKeys) => {
  console.log('Config changed:', changedKeys);
});

// Stop watching
config.unwatch();

// Cleanup
config.destroy();
```

**API Methods:**

- `get(key)` - Get value using dot notation (e.g., 'general.startMinimized')
- `set(key, value)` - Set value using dot notation
- `getAll()` - Get all config values (deep copy)
- `watch()` - Start file watching (500ms debounce)
- `unwatch()` - Stop file watching
- `save()` - Save to disk atomically (returns Promise)
- `reload()` - Reload from disk (returns array of changed keys)
- `validate()` - Validate against schema (returns {valid, errors})
- `destroy()` - Cleanup resources (watcher, timers, listeners)

**Events:**

- `configChanged` - Emitted when values change (params: changedKeys array)
- `error` - Emitted on errors (params: Error object)

**Error Handling:**

- Corrupt YAML files are backed up to `config.yaml.corrupt.{timestamp}`
- Write failures emit error events but preserve cache
- File locking prevents concurrent write conflicts (5s timeout)

### `migration.js`

SQLite to YAML migration system with version tracking and 30-day rollback period.

**Features:**

- One-time migration from SQLite to YAML
- Version tracking with migration date
- 30-day rollback period (SQLite settings preserved)
- Automatic backup creation
- Schema-based comment generation
- Comprehensive error handling

**Exports:**

- `migrateToYaml(configPath)` - Perform migration
- `isMigrationComplete()` - Check if migration done
- `getMigrationDate()` - Get migration date
- `isRollbackPeriodExpired()` - Check if rollback period passed
- `shouldMigrateSetting(key)` - Check if setting migrates to YAML
- `getMigratableSettings()` - Get list of YAML settings
- `getSqliteOnlySettings()` - Get list of SQLite-only settings
- `validateMigratedConfig(yamlConfig)` - Validate migrated config
- `flatToNested(flatSettings)` - Convert flat keys to nested
- Constants: `MIGRATION_MARKER`, `MIGRATION_DATE_KEY`, `SCHEMA_VERSION_KEY`

**Migration Example:**

```javascript
const { migrateToYaml, isMigrationComplete } = require('./migration');

if (!isMigrationComplete()) {
  const result = migrateToYaml('/path/to/config.yaml');

  if (result.success) {
    console.log(`Migrated ${result.migrated} settings`);
    // SQLite now has migration markers:
    // - settings_migrated_to_yaml: true
    // - settings_migration_date: ISO date
    // - config_schema_version: 2
  } else {
    console.error('Migration failed:', result.errors);
  }
}
```

See `migration.js` for full API documentation.

## Complete Example

```javascript
const path = require('path');
const { getConfigManager } = require('./manager');
const { migrateToYaml, isMigrationComplete } = require('./migration');

// On app startup
async function initializeConfig(appDataPath) {
  const configPath = path.join(appDataPath, 'config.yaml');

  // Migrate from SQLite if needed
  if (!isMigrationComplete()) {
    const result = migrateToYaml(configPath);
    if (result.success) {
      console.log(`Migrated ${result.migrated} settings to YAML`);
    }
  }

  // Get config manager singleton
  const config = getConfigManager(configPath);

  // Start watching for changes
  config.watch();
  config.on('configChanged', (keys) => {
    console.log('Settings changed:', keys);
    // Update UI, apply changes, etc.
  });

  return config;
}
```

## Settings Migration Rules

### Settings that go to YAML (user-editable):

- `general.*` (EXCEPT `general.startOnBoot` handled by installer)
- `appearance.*`
- `newNote.*`
- `tray.*`
- `shortcuts.*`
- `editor.*`
- `reminders.*`
- `history.*`
- `advanced.*`

### Settings that stay in SQLite:

- `whisper.*` - Complex state (model files, VAD settings)
- `setup.*` - One-time setup flags
- `hints.*` - UI hint state
- `general.startOnBoot` - Handled by installer registry entry

2. Update migration.js to handle old versions
3. Update this README with version history

## Validation

The schema enforces:

- **Type checking** - string, number, boolean
- **Range validation** - min/max for numbers
- **Enum validation** - allowed values for strings
- **Default values** - automatically applied for missing keys
- **Type coercion** - strings like "1" converted to numbers

## Usage in ConfigManager

```javascript
const { validateConfig } = require('./schema.js');
const { generateDefaultConfig } = require('./defaults.js');
const YAML = require('yaml');
const fs = require('fs');

// Load and validate config
const content = fs.readFileSync('config.yaml', 'utf8');
const parsed = YAML.parse(content);
const result = validateConfig(parsed);

if (!result.valid) {
  // Handle validation errors
  console.error('Invalid config:', result.errors);

  // Regenerate with defaults
  const defaultContent = generateDefaultConfig();
  fs.writeFileSync('config.yaml', defaultContent);
}
```
